AlgoQuantEngine
Quantitative portfolio analytics and risk modeling toolkit — Python
What it is

AlgoQuantEngine is a Python-based toolkit for quantitative portfolio analysis and risk evaluation. It is designed to apply structured, data-driven methods to financial datasets — computing performance metrics, modeling risk, and producing analytical outputs that reflect how a portfolio behaves over time.

What I built

Built end-to-end data pipelines for ingesting and processing financial time series data. Implemented quantitative risk metrics including volatility estimation, Sharpe ratio, and maximum drawdown. Designed a portfolio construction workflow that supports analysis of multi-asset compositions and benchmarking against reference indices. The project is structured as a reproducible command-line workflow with clean module separation between data ingestion, computation, and output.

Tech stack
Python pandas NumPy matplotlib Financial Data CLI
Key features
  • Time series ingestion and cleaning for financial data
  • Portfolio-level risk metric computation (Sharpe, volatility, drawdown)
  • Multi-asset portfolio analysis and benchmarking
  • Structured, reproducible command-line workflow
  • Analytical output generation for evaluation and reporting
What I learned

This project deepened my understanding of quantitative modeling and financial data engineering. Building the pipeline end-to-end — from raw data to structured analytical output — required careful attention to data quality, numerical precision, and reproducibility. It also pushed me to think more rigorously about how to structure a data-intensive Python project with clear module boundaries.

Skills shown: Data analysis Quantitative modeling Python engineering Analytical thinking

Blackjack AI Engine
Decision engine applying optimal strategy and expected value reasoning to Blackjack
What it is

A decision-making engine for Blackjack that evaluates game states and produces optimal actions based on probability theory and expected value analysis. The project explores how rule-based AI and mathematical reasoning can be applied to structured games — turning well-defined strategy into executable logic.

What I built

Implemented a game state evaluator that tracks cards, hand values, and dealer information. Developed decision logic based on standard Blackjack optimal strategy, extended with expected value calculations to handle edge cases. The engine can simulate game scenarios and evaluate the quality of decisions across sequences of hands.

Tech stack
Python Decision Logic Probability Simulation Expected Value
Key features
  • Game state representation and tracking
  • Optimal strategy implementation for standard Blackjack rules
  • Expected value computation for action selection
  • Simulation loop for evaluating strategy performance
What I learned

This project was an exercise in translating mathematical rules into a working software system. It strengthened my understanding of probability, decision trees, and how to structure AI logic cleanly. It also reinforced a broader lesson: intelligent behavior in a well-defined domain is largely a function of how rigorously you model the problem.

Skills shown: AI logic Probability reasoning Algorithm design Simulation

TeamTracker Application
Java application for managing team and player statistics with a structured OOP architecture
What it is

TeamTracker is a Java application designed for tracking sports team and player statistics. It emphasizes clean software architecture — clear separation of data models, application logic, and user interaction — and demonstrates the kind of structured, maintainable engineering that scales beyond a simple script.

What I built

Designed an object-oriented model with distinct classes for teams, players, and statistics, each with well-defined responsibilities. Built data management functionality for adding, updating, and removing records. Implemented persistence to allow data to be saved and reloaded across sessions. Developed a user interface layer that keeps display logic cleanly separated from the data model.

Tech stack
Java OOP GUI Data persistence MVC pattern
Key features
  • Structured class hierarchy for teams, players, and statistics
  • Full CRUD operations with data validation
  • Persistent storage — save and reload application data across sessions
  • User interface with clean separation from business logic
  • Maintainable, extensible code architecture
What I learned

This project was a hands-on exercise in building software that is designed to be maintained, not just to run. It reinforced the value of thinking in terms of responsibility — what each class should own, what it should not, and how pieces fit together. Working through persistence and interface integration also exposed the real complexity of building application-level software.

Skills shown: OOP design Java Data modeling Application architecture

DatePlanner App
iOS application for planning date courses and outings with a personalized, user-first design
What it is

DatePlanner is an iOS app designed to help users plan outings and date courses in a more thoughtful, structured way. Rather than simply searching for places, the app focuses on the experience of planning — organizing venues, routes, and preferences into a coherent outing that feels intentional rather than improvised.

What I built

Developed the app using SwiftUI with a focus on a clean and intuitive interface. Built planning flows that allow users to compose an outing from multiple locations in a defined sequence. Designed the data model to support saving, revisiting, and editing planned courses. Incorporated location-based thinking into the UX to make organization feel natural and reduce friction in the planning process.

Tech stack
Swift SwiftUI iOS UX Design MapKit
Key features
  • Outing course planning with ordered, editable location sequences
  • Saved course management for revisiting and reusing past plans
  • Clean, polished SwiftUI interface with smooth navigation
  • Personalization-oriented design focused on genuine user utility
What I learned

Building DatePlanner pushed me to think from a product perspective, not just an engineering one. The biggest challenge was not the code — it was deciding what the experience should feel like and then faithfully implementing that vision in SwiftUI. This project reinforced how much design thinking matters in making software that people actually want to use.

Skills shown: SwiftUI UX thinking Mobile development Product design

Weather Smart Planner
Activity planning tool that uses weather data to surface smarter scheduling decisions
What it is

Weather Smart Planner is a practical planning tool that integrates weather data to help users make better decisions about when and how to schedule outdoor or weather-sensitive activities. The goal is to turn forecast information into actionable planning guidance, rather than leaving that reasoning entirely to the user.

What I built

Built a data pipeline that fetches and processes weather forecast information from an external API. Implemented planning logic that maps weather conditions to activity suitability, allowing the application to produce recommendations or warnings based on the forecast. Designed a clean output interface that presents information in a way that is immediately useful for decision-making.

Tech stack
Python Weather API Data processing Planning logic
Key features
  • External weather API integration and data processing
  • Activity suitability scoring based on forecast conditions
  • Scheduling recommendations surfaced as actionable guidance
  • Clean, readable output designed for practical, everyday use
What I learned

This project was about making an external data source genuinely useful. Working with weather API data required careful handling of response formats, edge cases, and missing values. More broadly, it reinforced a principle I find valuable: the goal of software is to reduce friction for the user — to take something they would otherwise have to reason through themselves and surface the answer cleanly.

Skills shown: API integration Python Planning logic User-centered design
